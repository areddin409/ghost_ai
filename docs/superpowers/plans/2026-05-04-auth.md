# Feature 03: Auth Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire Clerk authentication into the Next.js 16 app with ClerkProvider, proxy-based route protection, styled sign-in/sign-up pages, and a UserButton in the editor navbar.

**Architecture:** `proxy.ts` at the project root (Next.js 16's renamed middleware) runs `clerkMiddleware` on every request, protecting all routes except the public sign-in and sign-up paths. `ClerkProvider` wraps the root layout using Clerk's dark base theme overridden with project CSS variables. Auth pages use a minimal two-panel layout (logo + features left, form right) that collapses to form-only on small screens.

**Tech Stack:** `@clerk/nextjs` v7, `@clerk/ui` (themes), Next.js 16 `proxy.ts` convention, React Server Components, Tailwind CSS tokens (no hardcoded colors)

## Task 2: Create proxy.ts

`proxy.ts` is Next.js 16's renamed `middleware.ts`. It exports a `proxy` function (not `middleware`). The Clerk handler is assigned to `proxy` so Next.js picks it up.

**Files:**
- Create: `proxy.ts`

- [ ] **Step 1: Write proxy.ts**

```ts
import { clerkMiddleware, createRouteMatcher } from '@clerk/nextjs/server'

const isPublicRoute = createRouteMatcher([
  `${process.env.NEXT_PUBLIC_CLERK_SIGN_IN_URL ?? '/sign-in'}(.*)`,
  `${process.env.NEXT_PUBLIC_CLERK_SIGN_UP_URL ?? '/sign-up'}(.*)`,
])

export const proxy = clerkMiddleware(async (auth, request) => {
  if (!isPublicRoute(request)) {
    await auth.protect()
  }
})

export const config = {
  matcher: ['/((?!_next/static|_next/image|favicon.ico|sitemap.xml|robots.txt).*)'],
}
```

## Task 3: Update app/layout.tsx — ClerkProvider + dark theme

Wrap the root layout with `ClerkProvider`. Apply Clerk's dark base theme and override color variables using the project's CSS custom properties. No hardcoded hex values.

**Files:**
- Modify: `app/layout.tsx`

- [ ] **Step 1: Update layout.tsx**

```tsx
import type { Metadata } from "next"
import { Geist, Geist_Mono } from "next/font/google"
import { ClerkProvider } from "@clerk/nextjs"
import { dark } from "@clerk/ui/themes"
import "./globals.css"

const geistSans = Geist({
  variable: "--font-geist-sans",
  subsets: ["latin"],
})

const geistMono = Geist_Mono({
  variable: "--font-geist-mono",
  subsets: ["latin"],
})

export const metadata: Metadata = {
  title: "Ghost AI",
  description: "Real-time collaborative system design workspace",
}

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode
}>) {
  return (
    <ClerkProvider
      dynamic
      appearance={{
        baseTheme: dark,
        variables: {
          colorBackground: "var(--bg-elevated)",
          colorPrimary: "var(--accent-primary)",
          colorText: "var(--text-primary)",
          colorTextSecondary: "var(--text-secondary)",
          colorInputBackground: "var(--bg-subtle)",
          colorInputText: "var(--text-primary)",
          colorDanger: "var(--state-error)",
        },
      }}
    >
      <html
        lang="en"
        className={`${geistSans.variable} ${geistMono.variable} h-full antialiased`}
      >
        <body className="min-h-full flex flex-col bg-background text-foreground">
          {children}
        </body>
      </html>
    </ClerkProvider>
  )
}
```

## Task 8: Update app/page.tsx — root redirect

Authenticated users go to `/editor`. Unauthenticated users go to `/sign-in`. Uses Clerk's `auth()` server function.

**Files:**
- Modify: `app/page.tsx`

- [ ] **Step 1: Update the file**

```tsx
import { auth } from "@clerk/nextjs/server"
import { redirect } from "next/navigation"

export default async function RootPage() {
  const { userId } = await auth()
  if (userId) redirect("/editor")
  redirect("/sign-in")
}
```
